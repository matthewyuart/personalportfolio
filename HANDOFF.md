## 1. CURRENT STATE & ARCHITECTURE

**Stack:** Next.js 15.5 (App Router) · React 19 · TypeScript · self-hosted Futura
(`next/font/local`) · one global stylesheet. No UI/CSS framework, no runtime
data, everything static-generated. `npm run build` passes clean → 12 static
pages, ~111 kB first-load JS.

**Routes**
- `/` — the whole site on one page: sections `#about → #projects → #art → #sketchbook → #photography`.
- `/work/[slug]` — 7 project detail pages (SSG). Slugs: `fashion-design, treehacks, stanfordshirt, 1525, sightline, canopycoffee, enose`.
- `/play` — experiments index (3 cards with thumbnails).
- `/play/oscillon`, `/play/hypercycles`, `/play/cultcube` — standalone static apps in `public/play/*` (each has an injected `<base>` tag + a floating "← play" back button).
- `/work` → 307-redirects to `/#projects` (config).

**Key files**
- `content/content.ts` — **single source of truth for ALL copy + data** (site chrome, about bio w/ inline links, 7 projects w/ verbatim copy, galleries `art`/`sketchbook`/`photography`, `experiments`, `sections`, `socials`). Every visible string lives here.
- `content/dims.json` — exact `[w,h]` for every image (generated by a sips loop). `pic(src)` in content.ts wires real dimensions into `next/image` so nothing renders cropped / with layout shift.
- `app/globals.css` — the entire design system (CSS variables + all component styles). One file on purpose.
- `app/layout.tsx` — self-hosts Futura Light/Book/Medium from `app/fonts/*.ttf` as `--font-futura`. Zero external font requests.
- `components/SiteNav.tsx` — shared header; "work ▾" is a CSS-only dropdown (hover + `:focus-within`) jumping to home sections.
- `components/Sketchbook.tsx` — client component; flippable stacked transparent-PNG pages (works; see issues).
- `components/Icons.tsx` — inline monochrome SVGs (instagram / x / linkedin / email), `currentColor`.
- `next.config.mjs` — `outputFileTracingRoot` pin (a stray `~/package-lock.json` otherwise breaks tracing), rewrites for the play apps, `/work`→`/#projects` redirect.

**Design decisions**
- Palette: cool light gray `--paper #e6e9ed`, ink `#1d1f22`. Tokens in `:root`.
- Type: Futura — **400 (regular) for body/readability, 300 (light) reserved for display** (name, eyebrow, headings). Curly quotes / `±` / `ü` render fine (FuturaCyrillic covers them).
- Images: **`next/image` everywhere** with explicit `width`/`height` from `dims.json`; sources pre-optimized to ≤1600px JPG (q80–82) via `sips`, so the client gets small WebP derivatives. Sketchbook pages stay **PNG** (transparency required).
- Galleries use **natural aspect ratios**: art = CSS `columns` masonry (no hover scale), photography = horizontal scroll-snap filmstrip, project images = 2-col masonry.
- Projects list has a **hover thumbnail** (`.row-thumb`).

**Media (all in `public/`, ~30 MB total)**
- `public/work/projects/<slug>/0N.jpg` — 17 images scraped verbatim from the old Framer site.
- `public/work/art/*` (10), `public/work/sketchbook/page-0N.png` (8, transparent), `public/work/photography/photo-0N.jpg` (9).
- `public/play/thumbs/{oscillon,hypercycles,cultcube}.jpg`, `public/headshot.jpg`.
- Originals were moved to `~/.Trash/portfolio-originals-*` (NOT deleted permanently) in case higher-res is needed.

## 3. CONSTRAINTS · DEPENDENCIES · ISSUES

**Hard constraints**
- Free hosting (Vercel). Keep it fast & light.
- **Project descriptions are verbatim** from matthewyu.dev — do NOT paraphrase/rewrite them.
- All editable copy stays in `content/content.ts`.

**Gotchas (learned the hard way)**
- ⚠️ **Never run `npm run build` while the dev server is running** — they share `.next` and the build clobbers the dev server's chunks (`Cannot find module './885.js'`). To type-check without touching `.next`, use `npx tsc --noEmit`. If it happens: stop dev, `rm -rf .next`, restart.
- After editing a client component, dev HMR sometimes wedges — a real `preview_stop` + `preview_start` (or kill + `npm run dev`) clears it.
- The preview screenshot tool times out on image-heavy pages (35 imgs) — that's tooling, not a site bug; verify via DOM/eval when it does.

## 5. STYLE · CONVENTIONS · STACK

- **Content-driven:** never hardcode copy/data in components — add it to `content/content.ts`. Images go through `pic("/path")` so dims come from `dims.json` (regenerate the map when adding images — small sips loop; see git history / ask).
- **One stylesheet** (`app/globals.css`), semantic class names (`.row`, `.masonry`, `.sb-page`, `.play-card`), CSS variables for theme. No Tailwind, no CSS-in-JS.
- **`next/image` always**, with explicit `width`+`height` (or `fill` in a ratio box) and a `sizes` hint. Never a bare `<img>` for content (sketchbook fold clone may be an exception).
- Futura weights: `300` display, `400` body. Cool-gray tokens only.
- Files: components in `components/*.tsx` (PascalCase), routes in `app/`. Keep components small and mostly presentational; the one client component is the Sketchbook.
- Verify with `npx tsc --noEmit` (never `npm run build` against a live dev server). Prod build only with dev stopped.
- **Image optimization recipe:** `sips -Z 1600 -s format jpeg -s formatOptions 80 in.ext --out out.jpg` (keep PNG for anything needing alpha). `cwebp`/`sips webp` are NOT available on this machine — rely on `next/image` for WebP delivery.
